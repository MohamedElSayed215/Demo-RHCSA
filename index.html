01Configure Network & HostnameNetwork►
  • IP: 172.25.250.11/24
  • Gateway + DNS: 172.25.250.254
  • Hostname: serverb.lab.example.com
1
Show connections — note the connection name
nmcli con show
2
Set static IP, gateway and DNS (replace "System eth0" with your connection name)
nmcli con mod "System eth0" ipv4.addresses 172.25.250.11/24 ipv4.gateway 172.25.250.254 ipv4.dns 172.25.250.254 ipv4.method manual
3
Apply and set hostname
nmcli con up "System eth0"
hostnamectl set-hostname serverb.lab.example.com
4
Verify
ip addr show eth0
hostnamectl
💡 ipv4.method manual = static IP. Without it the IP won't stick after reboot.
02Configure YUM RepositoryRepository►
  • File: /etc/yum.repos.d/rhel9.repo
  • Two sections: AppStream + BaseOS
  • gpgcheck=0
1
Open file in vi
vi /etc/yum.repos.d/rhel9.repo
2
Press i, paste this content, then save with :wq
[AppStream]
name=AppStream
baseurl=http://classroom.example.com/content/rhel9.0/x86_64/dvd/AppStream
enabled=1
gpgcheck=0

[BaseOS]
name=BaseOS
baseurl=http://classroom.example.com/content/rhel9.0/x86_64/dvd/BaseOS
enabled=1
gpgcheck=0
3
Verify both repos appear
dnf repolist
💡 gpgcheck=0 is required in the exam — no GPG key is available.
03SELinux — Allow Port 82 for httpdSELinux►
  • Allow SELinux for port 82/tcp
  • Change httpd Listen to 82
  • Open firewall + start httpd
1
Add port 82 to SELinux http context
semanage port -a -t http_port_t -p tcp 82
2
Change Listen port in httpd config
sed -i 's/Listen 80/Listen 82/' /etc/httpd/conf/httpd.conf
3
Open firewall and start httpd
firewall-cmd --permanent --add-port=82/tcp
firewall-cmd --reload
systemctl enable --now httpd
4
Verify port was added to SELinux
semanage port -l | grep http
⚠️ Never run setenforce 0 — you will lose marks. Fix SELinux properly with semanage.
04Create Users & GroupsUsers►
  • Group: admin
  • harry, natasha → secondary group: admin
  • sarah → shell: /sbin/nologin
  • Password for all: password
1
groupadd admin
2
useradd -G admin harry
useradd -G admin natasha
useradd -s /sbin/nologin sarah
3
Set passwords
echo 'password' | passwd --stdin harry
echo 'password' | passwd --stdin natasha
echo 'password' | passwd --stdin sarah
4
Verify
id harry
id sarah
💡 -G = supplementary group. -s /sbin/nologin = no interactive shell for sarah.
05Collaborative Directory /common/adminPermissions►
  • Directory: /common/admin
  • Group: admin
  • Permissions: 2770 (SGID)
1
mkdir -p /common/admin
chgrp admin /common/admin
chmod 2770 /common/admin
2
Verify — you should see drwxrws---
ls -ld /common/admin
💡 The 2 in 2770 = SGID. New files inside automatically inherit the admin group.
06Configure AutoFSAutoFS►
  • Install autofs
  • Auto-mount: /localhome/production5
  • Source: servera:/user-homes/production5
1
dnf install -y autofs
2
Add master map entry
echo "/localhome  /etc/auto.localhome" >> /etc/auto.master
3
Create the map file
echo "production5  -rw,sync  servera.lab.example.com:/user-homes/production5" > /etc/auto.localhome
4
systemctl enable --now autofs
💡 auto.master = where to mount. auto.localhome = what to mount and from where.
07Cron Job for harry — daily 12:30Cron►
  • User: harry
  • Time: every day at 12:30
  • Command: /bin/echo "hello"
1
Open harry's crontab editor
crontab -u harry -e
2
Add this line, then save with :wq
30 12 * * * /bin/echo "hello"
3
Verify
crontab -u harry -l
💡 Format: minute  hour  day  month  weekday. 30 12 * * * = 12:30 every day.
08Configure NTP (chronyd)NTP►
  • NTP server: classroom.example.com
  • Config file: /etc/chrony.conf
1
echo "server classroom.example.com iburst" >> /etc/chrony.conf
2
systemctl enable --now chronyd
3
Verify — look for ^* next to your server
chronyc sources
💡 RHEL 9 uses chrony, not ntpd. iburst = faster initial sync.
09Find Files Owned by sarahFind►
  • Find all files owned by user sarah
  • Copy them to /root/find.user/
1
mkdir /root/find.user
2
find / -user sarah -exec cp {} /root/find.user/ \; 2>/dev/null
3
ls /root/find.user/
💡 2>/dev/null hides "permission denied" errors from system directories.
10Search Text in /etc/passwdGrep►
  • Search for home in /etc/passwd
  • Save results to /root/search.txt
1
grep "home" /etc/passwd > /root/search.txt
2
cat /root/search.txt
💡 > saves output to file. >> appends instead of overwriting.

12Create Compressed Archive of /var/tmpArchive►
  • Compress /var/tmp
  • Save to /root/test.tar.gz
1
tar -czf /root/test.tar.gz /var/tmp
2
Verify archive contents
tar -tzf /root/test.tar.gz
💡 c=create  z=gzip  f=filename. The filename always comes right after f.
13Build Container ImageContainers►
  • Login: utility.lab.example.com  |  user: admin  |  pass: redhat
  • Build image named monitor
  • Do NOT modify the Containerfile
1
podman login utility.lab.example.com -u admin -p redhat
2
podman build -t monitor http://classroom.example.com/Containerfile
3
podman images
⚠️ Use podman — NOT docker. Docker is not installed in RHEL 9.
14Container as Systemd ServiceContainers►
  • Container: ascii2pdf from image monitor
  • /opt/files → /opt/incoming:Z
  • /opt/processed → /opt/outgoing:Z
  • Service: container-ascii2pdf starts on boot
1
Run the container
podman run -d --name ascii2pdf -v /opt/files:/opt/incoming:Z -v /opt/processed:/opt/outgoing:Z monitor
2
Create systemd service file
mkdir -p ~/.config/systemd/user
cd ~/.config/systemd/user
podman generate systemd --name ascii2pdf --files --new
3
Enable linger so service survives logout
loginctl enable-linger student
4
Enable and start the service
systemctl --user daemon-reload
systemctl --user enable --now container-ascii2pdf
5
systemctl --user status container-ascii2pdf
💡 Without loginctl enable-linger the container stops on logout. :Z on volumes = SELinux fix.
15Set umask for natashaPermissions►
  • New files = r-------- (400)
  • New dirs = r-x------ (500)
  • Permanent in /home/natasha/.bashrc
1
echo "umask 0266" >> /home/natasha/.bashrc
2
grep umask /home/natasha/.bashrc
💡 umask 0266 gives files=400, dirs=500. Adding to .bashrc makes it permanent.
16Password Expiry — 20 DaysSecurity►
  • Passwords for new users expire after 20 days
  • File: /etc/login.defs
1
sed -i 's/^PASS_MAX_DAYS.*/PASS_MAX_DAYS   20/' /etc/login.defs
2
grep PASS_MAX_DAYS /etc/login.defs
💡 This only affects new users. For existing users: chage -M 20 username
17Passwordless sudo for admin groupSudo►
  • Group admin can run any command without password
  • File: /etc/sudoers.d/admin
1
echo "%admin ALL=(ALL) NOPASSWD: ALL" > /etc/sudoers.d/admin
2
chmod 440 /etc/sudoers.d/admin
3
Validate — no errors = correct
visudo -c
💡 %admin = group. chmod 440 is required — wrong permissions = visudo rejects the file.

18Create LVM Logical VolumeLVM►
  • PV: /dev/sdc
  • VG: datastore — PE size 8 MiB
  • LV: database — 50 extents
  • Filesystem: ext3
  • Mount: /mnt/database (permanent)
1
pvcreate /dev/sdc
2
vgcreate -s 8MiB datastore /dev/sdc
3
lvcreate -l 50 -n database datastore
4
mkfs.ext3 /dev/datastore/database
5
Mount permanently via fstab
mkdir -p /mnt/database
echo '/dev/datastore/database /mnt/database ext3 defaults 0 0' >> /etc/fstab
mount -a
6
df -h /mnt/database
💡 Order always: pvcreate → vgcreate → lvcreate → mkfs → fstab → mount.
-l 50 = 50 extents = 50 × 8 MiB = 400 MiB total.
19Extend LV to 100 ExtentsLVM►
  • Extend database LV to 100 extents total
  • Resize the filesystem to use the new space
1
lvextend -l 100 /dev/datastore/database
2
Unmount, resize filesystem, remount
umount /mnt/database
resize2fs /dev/datastore/database
mount -a
3
df -h /mnt/database
💡 ext3/ext4 → resize2fs (needs unmount first).  XFS → xfs_growfs /mnt/database (no unmount needed).
20Configure Tuned ProfileTuned►
  • Install and enable tuned
  • Apply the recommended profile
1
dnf install -y tuned

2
Get recommendation and apply it
tuned-adm recommend
tuned-adm profile virtual-guest

💡 Apply whatever tuned-adm recommend outputs — in a VM it's usually virtual-guest.
